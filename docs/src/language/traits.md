# Traits

Traits define shared behavior that types can implement. They are SpiteScript's
mechanism for polymorphism -- a function can accept any type that implements a given
trait, without knowing the concrete type at the definition site.

## Declaring a Trait

A trait declares a set of method signatures that implementing types must provide.

```
trait Describable {
    fn describe(&self) -> String;
}

trait Comparable {
    fn compare(&self, other: &Self) -> i32;
}
```

## Default Implementations

Traits can provide default method bodies. Implementing types inherit the default
unless they override it.

```
trait Printable {
    fn to_string(&self) -> String;

    fn print(&self) {
        print(self.to_string());
    }
}
```

Types that implement `Printable` must provide `to_string`, but get `print` for free:

```
impl Printable for Point {
    fn to_string(&self) -> String {
        return `(${self.x}, ${self.y})`;
    }
    // print() is inherited from the default
}

let p = Point { x: 3.0, y: 4.0 };
p.print();    // prints "(3.0, 4.0)"
```

## Implementing Traits for Types

Use `impl TraitName for TypeName` to implement a trait.

```
struct Point {
    x: f64,
    y: f64,
}

struct User {
    id:   u64,
    name: String,
}

impl Describable for Point {
    fn describe(&self) -> String {
        return `Point(${self.x}, ${self.y})`;
    }
}

impl Describable for User {
    fn describe(&self) -> String {
        return `User(${self.id}: ${self.name})`;
    }
}
```

Now both `Point` and `User` can be passed to any function expecting a `Describable`.

## Trait Bounds on Generics

Use trait bounds to constrain generic type parameters. A bound ensures the type
parameter has the methods required by the function body.

Single bound:

```
fn print_desc<T: Describable>(item: T) {
    print(item.describe());
}

print_desc(Point { x: 1.0, y: 2.0 });    // "Point(1.0, 2.0)"
print_desc(User { id: 1, name: "Alice" }); // "User(1: Alice)"
```

Multiple bounds with `+`:

```
fn describe_and_compare<T: Describable + Comparable>(a: T, b: T) -> String {
    let cmp = if a.compare(&b) > 0 { "greater" } else { "lesser" };
    return `${a.describe()} is ${cmp} than ${b.describe()}`;
}
```

Bounds work on any generic position -- functions, structs, and impl blocks:

```
fn largest<T: Comparable>(a: T, b: T) -> T {
    if a.compare(&b) > 0 {
        return a;
    }
    return b;
}
```

## Built-in Traits

SpiteScript provides these built-in traits in the standard library:

| Trait | Description |
|-------|-------------|
| `Clone` | Deep copy via `.clone()` |
| `Eq` | Equality comparison with `==` and `!=` |
| `Hash` | Hashing for use as `Map` keys |
| `Comparable` | Ordering with `<`, `>`, `<=`, `>=`, `<=>` |
| `Display` | String representation via `.to_string()` and string interpolation |
| `Debug` | Debug-formatted string representation |
| `Error` | Error trait, auto-implemented by `@error` |
| `From<T>` | Type conversion from `T`, generated by `@from` |
| `Iterator` | Enables use in `for` loops and all pipeline operators |

### The `Iterator` Trait

Any type can be made iterable by implementing `Iterator`:

```
trait Iterator {
    type Item;
    fn next(&mut self) -> Option<Self::Item>;
}
```

Once implemented, the type works in `for` loops and supports all pipeline operators
(`map`, `filter`, `fold`, etc.).

## The `@derive` Attribute

The `@derive` attribute auto-generates trait implementations for structs and enums.
This eliminates boilerplate for common traits.

```
@derive(Clone, Eq, Hash, Debug)
struct Point {
    x: i32,
    y: i32,
}
```

This generates implementations of `Clone`, `Eq`, `Hash`, and `Debug` for `Point`
based on its fields.

Derivable traits and their behavior:

| Trait | Generated behavior |
|-------|-------------------|
| `Clone` | Deep-copies each field |
| `Eq` | Field-by-field equality |
| `Hash` | Hashes each field in declaration order |
| `Comparable` | Lexicographic comparison on fields in declaration order |
| `Debug` | Formats as `TypeName { field: value, ... }` |
| `Display` | Same format as `Debug` |

You can derive multiple traits in one attribute:

```
@derive(Clone, Eq, Hash, Comparable, Debug, Display)
enum Priority {
    Low,
    Medium,
    High,
}
```

For enums, the derived `Comparable` orders variants by their declaration position
(e.g. `Low < Medium < High`).

## Combining Traits and Generics

Traits and generics work together to create reusable, type-safe abstractions.

```
@derive(Clone, Eq, Debug)
struct Pair<A, B> {
    first:  A,
    second: B,
}

fn swap<A: Clone, B: Clone>(p: Pair<A, B>) -> Pair<B, A> {
    return Pair { first: p.second.clone(), second: p.first.clone() };
}

let original = Pair { first: 1, second: "hello" };
let swapped = swap(original);
// swapped.first = "hello", swapped.second = 1
```

## Complete Example

```
trait Area {
    fn area(&self) -> f64;
}

trait Perimeter {
    fn perimeter(&self) -> f64;
}

struct Circle { radius: f64 }
struct Rect   { width: f64, height: f64 }

impl Area for Circle {
    fn area(&self) -> f64 {
        return 3.14159 * self.radius * self.radius;
    }
}

impl Perimeter for Circle {
    fn perimeter(&self) -> f64 {
        return 2.0 * 3.14159 * self.radius;
    }
}

impl Area for Rect {
    fn area(&self) -> f64 {
        return self.width * self.height;
    }
}

impl Perimeter for Rect {
    fn perimeter(&self) -> f64 {
        return 2.0 * (self.width + self.height);
    }
}

fn describe_shape<T: Area + Perimeter + Describable>(shape: T) {
    print(`${shape.describe()}: area=${shape.area()}, perimeter=${shape.perimeter()}`);
}
```
